Operating Systems Terminology
TERM DEFINITION
Operating System The software that controls the overall operation of a computer, it allows you to execute programs and to store/take files.
Examples:
  • Windows
  • UNIX
    • Mac OS
    • Solaris
  • Linux
Job The program to be executed.
Job Queue The jobs residing in mass storage that wait for execution.
FIFO Short for: First In, First Out
The process where a job is processed and removed from a queue. First one in goes first and out first, but some systems allow jobs to take priority over others.
Interactive Processing Using a remote terminal to carry on dialogue with the user for a program being executed. Its a way to interact while a program is being executed.
Examples
  • Reservation Systems
  • Cancellations
  • Word Processing Systems
Time-sharing Providing service to multiple users at the same time using multi-programming or another method- like multitasking.
Multi-programming A technique for time-sharing that divides time into intervals which restricts executions of jobs to one interval at a time. After one interval is finished another job is given an interval to execute, and then after that another and so on.
Application Software Consist of programs that perform tasks particular to the machineâ€™s utilization. The type of program it is can vary depending on what it is being utilized for.
Examples:
  • Spreadsheets
  • Desktop Publishing Systems
  • Program Development Software
System Software Performs tasks that are common to computer systems in general. Application software require this as a base (like infrastructure) so that it may function.
There are two types:
  • The Operating System
  • Utility Software
Utility Software A collective of software units that perform activities fundamental to computer installations. Software that extend or customize the capabilities of the operating system. It does not come with the operating system.
Examples(Software that can):
  • Compress/decompress data
  • Play multimedia content
  • Handle network communication
GUI Short for: Graphical User Interface
Objects that can be manipulated are represented with images, pictorially, on display icons (ex: files and programs). These allow users to issue commands using a common input devices (ex: mouse, keyboard, or touch-pad).
Shell Communicate with users through textual messages using a keyboard and monitor screen. Allow users to issue commands using text.
Kernel The internal part of an operating system. It contains software components that perform the very basic functions required by installation.
Units in the Kernel:
  • Device Driver
  • File Manager
  • Memory Manager
  • Scheduler
  • Dispatcher

File Manager A unit in the Kernal. Its job is to coordinate the machineâ€™s mass storage spaces/facilities. Ex(It decided/knows): Where a file is located, which users can access it, and how much space is available.
Memory Manager A unit in the Kernal. Coordinates the machineâ€™s use of main memory. It finds and assigns memory space for programs and blocks of data to unsure that they donâ€™t go over the programâ€™s limited space.
Virtual Memory Fake additional memory space which is achieved by using a technique called paging. This space is created by rotating programs and data back and forth between main memory and mass storage.
Boot Strapping Performed every time a computer is turned on. A boot loader transfers(copies) the operating system from mass storage and into main memory.
ROM Short for: Read-Only Memory
Nonvolatile memory- it does not get erased after a shut down. Contents can be read but not altered. The boot loader is permanently stored in here.
Deadlock A condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another. The processes are eternally locked waiting for the other to move.
Spooling A technique used to hold data to output at a later but more convenient time. This grants multiple processes access to a common resource.
Example:
A file manager can allow multiple processes access to read a file (file does not change), but having multiple change a file would create conflict, which would then require allocating according to need.
Super User (or Administrator) This is an account(a login) that allows a person highly privileged access to the operating system. They can modify parts of the operating system and monitor activities going on in the system.
Multi-Factor Authentication Instead of only using a password, this requires multiple pieces of evidence or steps before authenticating the user.
